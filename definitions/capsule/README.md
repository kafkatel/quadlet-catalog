# Red Hat Satellite Capsule Server (Experimental)

⚠️ **EXPERIMENTAL/LAB-ONLY:** This is not an official Red Hat Satellite container deployment method. Satellite 6.19 (target: May 2026) will ship official containerized Capsule images. This definition creates a UBI-init systemd container for lab/testing only.

For production deployments, use a full RHEL installation with the [parmstro/rhis-builder-satellite](https://github.com/parmstro/rhis-builder-satellite) Ansible playbooks (capsule roles).

---

A Satellite Capsule is a distributed Smart Proxy that mirrors content from the main Satellite server, providing content delivery, provisioning, and configuration management to remote locations or network segments.

This quadlet deploys a Capsule as a systemd container with its own LAN IP via bridge networking.

## Stack Components

| Service | Image | Description |
|---------|-------|-------------|
| capsule | `localhost/capsule:latest` (locally built) | Red Hat Satellite Capsule |

## Architecture

Standalone systemd container on an unmanaged bridge network. The Capsule registers with a parent Satellite server and synchronizes content from it.

```
┌───────────────────────────────────────────────────┐
│  Host LAN (192.168.1.0/24)                       │
│                                                   │
│  satellite (192.168.1.101) ←──┐                 │
│                                 │                 │
│  capsule (192.168.1.102) ──────┘                │
│     └─ systemd (PID 1)                           │
│        ├─ Smart Proxy (foreman-proxy)            │
│        ├─ Pulp (content cache)                   │
│        └─ Apache (httpd)                         │
└───────────────────────────────────────────────────┘
```

## Quadlet Files

| File | Type | Purpose |
|------|------|---------|

## Prerequisites

- **OS:** RHEL 9, Fedora 38+, or CentOS Stream 9
- **Podman:** 4.4+ (rootful)
- **Resources:**
  - **RAM:** 8 GB minimum, 16 GB recommended
  - **CPU:** 2 cores minimum
  - **Disk:** 50 GB minimum (grows with synced content)
- **Bridge networking:** Host must have br0 configured. See [docs/bridge-networking.md](../../docs/bridge-networking.md).
- **Parent Satellite:** A running Satellite server that this Capsule will register with
- **Red Hat subscription:** Required for Capsule packages

## Setup

### 1. Bridge Networking

See [docs/bridge-networking.md](../../docs/bridge-networking.md).

### 2. Register Build Host

```bash
sudo subscription-manager register --username <your-rh-username>
sudo subscription-manager attach --auto
sudo subscription-manager repos --enable=satellite-6.18-for-rhel-9-x86_64-rpms
sudo subscription-manager repos --enable=satellite-capsule-6.18-for-rhel-9-x86_64-rpms
```

### 3. Generate Capsule Certificates on the Satellite

**⚠️ CRITICAL STEP:** Capsules require certificates generated by the parent Satellite. On your Satellite server:

```bash
# Generate certs for the Capsule
sudo capsule-certs-generate \
    --foreman-proxy-fqdn capsule1.example.com \
    --certs-tar /var/www/html/pub/capsule1.example.com-certs.tar
```

This creates a tarball containing:
- SSL certificates signed by Satellite's CA
- OAuth consumer key and secret
- Katello CA certificate

**Download the tarball to the Capsule host:**

```bash
# From the Capsule host:
curl -k -O https://satellite1.example.com/pub/capsule1.example.com-certs.tar

# Or use scp:
scp satellite1.example.com:/var/www/html/pub/capsule1.example.com-certs.tar .
```

Keep this tarball secure — it contains credentials for this Capsule.

### 4. Create Host Directories

```bash
sudo mkdir -p /srv/containers/capsule/{pulp,foreman-proxy,puppet-ssl,log,certs}

# Copy the certs tarball into the certs directory
sudo cp capsule1.example.com-certs.tar /srv/containers/capsule/certs/
```

### 5. Configure Network Subnet

Edit `capsule-bridge.network`:

```ini
Subnet=192.168.1.0/24
Gateway=192.168.1.1
IPRange=192.168.1.100/28
```

### 6. Configure Capsule Settings

Edit `capsule.env`:

```ini
CAPSULE_HOSTNAME=capsule1.example.com
SATELLITE_HOSTNAME=satellite1.example.com
CAPSULE_ORGANIZATION=My_Org
CAPSULE_LOCATION=Remote_Office
```

### 7. Build the Container Image

```bash
cd definitions/capsule/
sudo podman build -t localhost/capsule:latest -f Containerfile .
```

### 8. Install Quadlet Files

```bash
sudo cp *.build *.container *.network *.volume *.env /etc/containers/systemd/
sudo systemctl daemon-reload
```

### 9. Start the Container

```bash
sudo systemctl start capsule.service
```

### 10. Install Capsule Certificates

Attach to the container and extract the certs tarball:

```bash
sudo podman exec -it capsule bash
```

Inside the container:

```bash
# Extract certs to the filesystem
tar xf /var/lib/pulp/capsule1.example.com-certs.tar -C /

# Verify certs were extracted
ls -l /etc/pki/katello-certs-tools/
```

### 11. Run capsule-installer

Still inside the container:

```bash
# Interactive mode
satellite-installer --scenario capsule

# Or unattended mode
satellite-installer --scenario capsule \
    --foreman-proxy-register-in-foreman true \
    --foreman-proxy-foreman-base-url "https://satellite1.example.com" \
    --foreman-proxy-trusted-hosts satellite1.example.com \
    --foreman-proxy-oauth-consumer-key "<from-certs-tar>" \
    --foreman-proxy-oauth-consumer-secret "<from-certs-tar>" \
    --enable-foreman-proxy-plugin-discovery \
    --enable-foreman-proxy-plugin-ansible \
    --enable-foreman-proxy-plugin-remote-execution-script \
    --foreman-proxy-content-parent-fqdn satellite1.example.com \
    --foreman-proxy-content-pulp-oauth-secret "<from-certs-tar>"
```

The OAuth keys are in the extracted certs. Check:

```bash
grep oauth /root/capsule-certs-generate-output.txt
```

Or use the interactive installer — it reads them automatically from the extracted certs.

### 12. Verify

```bash
# Check services inside the container
sudo podman exec capsule systemctl status foreman-proxy

# Verify registration with Satellite
# On the Satellite server:
sudo podman exec satellite hammer capsule list
# Should show capsule1.example.com
```

## Day-2 Operations

### Syncing Content to the Capsule

Content sync is triggered from the Satellite, not the Capsule. On the Satellite:

```bash
# List Capsules
sudo podman exec satellite hammer capsule list

# Start content sync
sudo podman exec satellite hammer capsule content synchronize \
    --name capsule1.example.com
```

Or via the Satellite web UI: Infrastructure → Capsules → Select Capsule → Actions → Synchronize

### Viewing Capsule Logs

```bash
# Systemd journal
sudo journalctl -u capsule.service -f

# Foreman Proxy log inside container
sudo podman exec capsule tail -f /var/log/foreman-proxy/proxy.log

# Persistent logs on host
sudo tail -f /srv/containers/capsule/log/foreman-proxy/proxy.log
```

### Updating Capsule Certificates

If certificates expire or need to be regenerated:

1. Generate new certs on the Satellite (same `capsule-certs-generate` command)
2. Download the new tarball
3. Stop the Capsule: `sudo systemctl stop capsule.service`
4. Copy new tarball: `sudo cp new-certs.tar /srv/containers/capsule/certs/`
5. Start Capsule: `sudo systemctl start capsule.service`
6. Extract new certs and re-run installer inside the container

## Data Persistence

| Host Path | Container Mount | Purpose | Typical Size |
|-----------|----------------|---------|--------------|
| `/srv/containers/capsule/pulp` | `/var/lib/pulp` | Synced content from Satellite | 50-200+ GB |
| `/srv/containers/capsule/foreman-proxy` | `/etc/foreman-proxy` | Smart Proxy configuration | < 100 MB |
| `/srv/containers/capsule/puppet-ssl` | `/etc/puppetlabs/puppet/ssl` | Puppet SSL certificates | < 100 MB |
| `/srv/containers/capsule/log` | `/var/log` | All service logs | 1-10 GB |

## Port Reference

| Port | Protocol | Service | Notes |
|------|----------|---------|-------|
| 443 | TCP | HTTPS | Web UI, API, content delivery |
| 5647 | TCP | qpid | Client communication (remote execution) |
| 8000 | TCP | Proxy | Smart Proxy API |
| 8140 | TCP | Puppet | Puppet agent check-ins |
| 9090 | TCP | Cockpit | Optional web console |
| 69 | UDP | TFTP | PXE boot files (if enabled) |
| 53 | TCP/UDP | DNS | If Smart Proxy DNS is enabled |
| 67 | UDP | DHCP | If Smart Proxy DHCP is enabled |

## Certificate Generation Workflow

Capsules require certificates from the parent Satellite for secure communication.

### On the Satellite

```bash
# Generate certs
sudo capsule-certs-generate \
    --foreman-proxy-fqdn capsule1.example.com \
    --certs-tar /var/www/html/pub/capsule1-certs.tar

# The output includes the OAuth consumer key and secret
# Save this output -- you'll need it for the installer
```

### Transfer to Capsule

```bash
# Make the tarball web-accessible
sudo chmod 644 /var/www/html/pub/capsule1-certs.tar

# Download from Capsule host
curl -k -O https://satellite1.example.com/pub/capsule1-certs.tar

# Or use scp
scp satellite1.example.com:/var/www/html/pub/capsule1-certs.tar \
    /srv/containers/capsule/certs/
```

### Inside the Capsule Container

```bash
sudo podman exec -it capsule bash

# Extract to the filesystem
tar xf /var/lib/pulp/capsule1-certs.tar -C /

# Verify extraction
ls -l /etc/pki/katello-certs-tools/
```

The extracted certs include:
- `/etc/pki/katello-certs-tools/katello-server-ca.crt` — Satellite CA
- `/etc/pki/katello/certs/katello-apache.crt` — Capsule cert
- `/etc/pki/katello/private/katello-apache.key` — Capsule key
- OAuth credentials in the installer output

## Troubleshooting

### "Unable to authenticate with Satellite" during installer

Verify:
1. Certs tarball was extracted to `/` (not a subdirectory)
2. OAuth consumer key/secret match the Satellite's output
3. Capsule can reach Satellite on port 443: `curl -k https://satellite1.example.com/`
4. Hostname matches the FQDN used in `capsule-certs-generate`

### Content sync fails

Check:
1. Capsule is registered: `hammer capsule list` on Satellite
2. Capsule services are running: `systemctl status foreman-proxy` inside container
3. Network connectivity to Satellite on port 5647
4. Disk space available in `/srv/containers/capsule/pulp`

### "foreman-proxy failed to start"

Check Smart Proxy logs:

```bash
sudo podman exec capsule journalctl -u foreman-proxy -n 100
```

Common issues:
- Missing certs or wrong paths
- Incorrect OAuth credentials
- Satellite hostname not resolvable
- Port conflicts (another service using 8000 or 8140)

## Security Considerations

- **Certs tarball** — contains credentials; delete after extraction or secure it
- **OAuth secrets** — treat like passwords
- **No privileged mode** — test without `--privileged` first; only add if installer fails
- **TLS certificates** — generated by Satellite, signed by Satellite's CA
